第五阶段：实现 Supabase 云端照片上传和相册系统

接下来才做照片系统。发：

现在请实现 Supabase 云端照片上传和相册系统。

重要原则：

1\. 项目初始状态下，所有用户账户下的所有国家、城市、地标相册都是空的。

2\. 不要预置任何真实照片。

3\. 不要创建 samplePhotos.ts。

4\. 用户照片必须上传到 Supabase Storage。

5\. 照片元数据必须保存到 Supabase Database。

6\. 每张照片必须绑定 userId。

7\. 用户只能看到和管理自己的照片。

具体要求：

1\. 创建 photos 数据表。

2\. 创建 user\_places、visit\_records、albums 数据表。

3\. 创建 Supabase Storage bucket。

4\. 配置 Row Level Security。

5\. 实现照片上传到 Supabase Storage。

6\. 上传照片时必须填写：

&#x20;  - 照片文件

&#x20;  - 照片标题

&#x20;  - 拍摄日期：年月日，必填

&#x20;  - 所属国家

7\. 上传照片时可选填写：

&#x20;  - 拍摄时间

&#x20;  - 所属城市

&#x20;  - 所属地点 / 地标

&#x20;  - 备注

&#x20;  - 标签

8\. 上传成功后，把照片元数据写入 photos 表。

9\. 保存 fileUrl、filePath、fileName、title、countryId、cityId、landmarkId、takenDate、takenTime、uploadedAt、notes、tags、visibility。

10\. 根据 userId 读取当前用户自己的照片。

11\. 空相册显示可爱空状态。

12\. 上传第一张照片后，相册图标从空状态变成有照片状态。

13\. 页面刷新后，从 Supabase 重新加载照片和旅行记录。

14\. 删除照片时，需要同时删除 Supabase Storage 文件和 photos 数据表记录。

15\. 用户 A 不能看到、修改或删除用户 B 的照片。

这一阶段是最关键的，因为它决定你的网站是不是真实网站而非Local host。
